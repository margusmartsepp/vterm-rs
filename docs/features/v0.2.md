# v0.2 — eyes, hands, lifecycle

Transformed the terminal host from a simple pipe into a state-aware, interactive
orchestration platform. Implemented a full virtual terminal state (`vt100`) and native
Windows window management.

## Skills

### `screen_read` — the eyes

Capture the current visual state of the terminal grid without screen-grabbing the OS.
A `vt100` parser maintains an in-memory 80×30 character grid; output is the clean text
the user actually sees. Lets the AI verify command completion, detect errors, and read
streaming output (`ping`, `top`, etc.).

### `screen_write` — the hands

Send text and keystroke shortcuts to the PTY. Recognised tokens:

| Token         | Bytes sent              | Use                         |
| ------------- | ----------------------- | --------------------------- |
| `<C-c>`       | `0x03`                  | SIGINT — interrupt          |
| `<C-d>`       | `0x04`                  | EOF                         |
| `<Tab>`       | `0x09`                  | Completion                  |
| `<Enter>`     | `\r\n`                  | Newline                     |
| `<Up>`/`<Down>`/`<Left>`/`<Right>` | CSI `\x1b[A`/`B`/`C`/`D` | Cursor      |

### `screen_control` — the lifecycle

Direct manipulation of the spawned terminal window via Win32:
`pin`, `unpin`, `minimize`, `maximize`, `restore`, `menu`, `close`.

### `screen_close` — the cleanup

Targeted (`id`) or bulk (`target: "all"`) termination of sessions.

## Destructors

- **Time-based**: terminate after a duration. Prevents abandoned long-runners.
- **Line-count-based**: terminate after producing too much output. Catches runaway
  loops before they freeze the UI.
- A background watchdog polls every 500 ms and triggers a graceful close.
