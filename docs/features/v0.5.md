# v0.5 — supreme (planned, partially shipped)

The v0.5 series introduced the architectural pillars for deterministic synchronisation,
high-fidelity observability, and resource management. Several features were *specified*
but not fully implemented; v0.6 closes that gap. This document describes the **intent**
of v0.5; see [v0.6.md](v0.6.md) for the actual delivery.

## Pillars

### 1. Correlation ID protocol

Every request supports an optional `req_id: u64`. Server echoes it on every response.
Eliminates positional response counting on the client.

### 2. Atomic batch aggregation

`Batch` returns one response with `sub_results: Vec<CommandResult>`. Top-level `status`
reflects aggregate success even when `stop_on_error = false`.

### 3. Session-based reaping

Every connection is assigned a `connection_id`. On disconnect, every PTY/process owned
by that connection is killed and reaped. No zombie `powershell.exe` accumulation.

### 4. PID → HWND mapping

Window control uses `EnumWindows` + `GetWindowThreadProcessId` to find the exact HWND of
the spawned process. Survives title changes (`cd`, custom prompts).

### 5. Smart prompt handshake

Replaces `sleep(2000)` with a `WaitUntil(pattern = prompt_regex)` before injecting any
initial command. Faster on fast machines, reliable on slow ones.

### 6. Structured observability

Replaces `eprintln!` with `tracing` spans, one per terminal interaction.
